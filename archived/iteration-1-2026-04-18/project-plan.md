# Project Plan: tiny-wiki

## 規劃摘要
- **目標**: 建立一個基於 Node.js + Vue 3 的本地 Wiki 系統，支援多層級目錄、Markdown 編輯與即時預覽。
- **假設**: 本地開發環境已安裝 Node.js 18+；Repository 路徑將透過環境變數或啟動參數傳遞。
- **風險**: 
    - 檔案操作的權限問題。
    - 大量檔案或深層目錄時的 API 回傳效能。
    - 檔案系統的即時同步（若手動在外部修改檔案）。

## 任務清單

### Task 1 - 專案基礎腳手架與環境設定
- **為什麼現在做**: 建立開發的基礎架構，確保前後端能協同運作。
- **目標**: 初始化 Node.js 後端與 Vue 3 前端，並設定開發模式的 Proxy。
- **背景 / 依賴**: 無。
- **粗略作法**: 
    1. 建立 `server/` (Node.js) 與 `client/` (Vue 3 + Vite) 資料夾。
    2. 初始化 `npm` 專案，安裝 `express`, `cors` 等基本後端套件。
    3. 設定 Vite 代理，將 `/api` 請求轉發至後端。
- **驗證方式**: 執行 `npm run dev` 能同時啟動前後端，且前端能成功請求一個測試 API 的 "Hello World"。
- **風險 / 備註**: 確保開發時的埠號不衝突。

### Task 2 - 後端 - 檔案樹檢索 (File Tree API)
- **為什麼現在做**: 導覽是 Wiki 的核心，後續所有操作都依賴於檔案結構。
- **目標**: 實作一個 API，能遞迴掃描指定 Folder 並回傳嵌套的資料夾/檔案 JSON 結構。
- **背景 / 依賴**: Task 1 已完成。
- **粗略作法**: 
    1. 使用 Node.js 的 `fs.readdirSync` 或 `fs.promises` 模組。
    2. 過濾掉 `.git`, `node_modules` 等不相關目錄。
    3. 實作 `GET /api/tree` 接口。
- **驗證方式**: 使用 `curl` 或 Postman 請求 `/api/tree`，回應的 JSON 應正確反映磁碟上的多層級目錄結構。
- **風險 / 備註**: 遞迴寬度與深度若太大可能影響效能，初期實作基本遞迴即可。

### Task 3 - Frontend - 側邊欄檔案導覽元件
- **為什麼現在做**: 讓使用者能看見並切換不同頁面。
- **目標**: 在 Vue 3 中實作一個樹狀導覽組件，渲染 Task 2 的 API 資料。
- **背景 / 依賴**: Task 2 已完成。
- **粗略作法**: 
    1. 建立 `FileTree.vue` 遞迴組件。
    2. 使用 Vue 狀態管理（或簡單的 Reactive object）存儲 tree 資料。
    3. 點擊檔案時變更「當前選取檔案」的路徑。
- **驗證方式**: 前端介面能正確顯示 Repository 的目錄，且點擊資料夾能收合/展開。
- **風險 / 備註**: 樣式需美觀，確保選取狀態有清晰標示。

### Task 4 - 後端 - 檔案內容讀寫與刪除 API
- **為什麼現在做**: 實現 Wiki 內容的持久化。
- **目標**: 建立讀取 (`GET`)、寫入 (`POST`/`PUT`) 及刪除 (`DELETE`) 實體 Markdown 檔案的 API。
- **背景 / 依賴**: Task 2 設定的 Repository 路徑。
- **粗略作法**: 
    1. 實作 `GET /api/file?path=...` 回傳檔案字串。
    2. 實作 `POST /api/file` 接收路徑與內容並儲存。
    3. 實作 `DELETE /api/file` 刪除檔案或空資料夾。
- **驗證方式**: 透過 API 測試軟體確認檔案能被讀取，修改後的內容能正確寫入磁碟，且刪除操作確實生效。
- **風險 / 備註**: 路徑安全檢查（避免 Path Traversal 攻擊）。

### Task 5 - 前端 - 閱讀模式 (Viewer)
- **為什麼現在做**: 使用者首先需要能看見筆記內容。
- **目標**: 根據選取的路徑讀取內容，並使用 Markdown 庫渲染成 HTML。
- **背景 / 依賴**: Task 4 的讀取 API 已完成。
- **粗略作法**: 
    1. 引入 `markdown-it` 套件。
    2. 實作 `ArticleView.vue` 元件。
    3. 整合代碼高亮插件 (如 `highlight.js`)。
- **驗證方式**: 點擊側邊欄檔案後，右側主區域能正確顯示渲染後的 HTML，且樣式符合預期。
- **風險 / 備註**: Markdown 渲染後的樣式 (CSS) 需要針對標題、代碼塊進行美化。

### Task 6 - 前端 - 編輯模式與即時預覽 (Editor)
- **為什麼現在做**: 本專案的核心編輯功能。
- **目標**: 提供切換按鈕，進入「編輯模式」後顯示雙欄位編輯與預覽。
- **背景 / 依賴**: Task 5 的渲染邏輯。
- **粗略作法**: 
    1. 使用 `<textarea>` 作為輸入框。
    2. 編輯時同步更新內容到渲染區。
    3. 實作「儲存」按鈕呼叫 Task 4 的寫入 API。
- **驗證方式**: 進入編輯模式，輸入 Markdown 時預覽區即時更新，點擊儲存後重新整理頁面，內容維持不變。
- **風險 / 備註**: 編輯到一半切換頁面時的「未儲存提示」。

### Task 7 - 檔案生命週期管理 (新增與刪除功能)
- **為什麼現在做**: 完善 Wiki 的動態管理能力。
- **目標**: 在 UI 上實作新增檔案、新增資料夾及刪除現有檔案的功能。
- **背景 / 依賴**: 前述所有 API 與元件。
- **粗略作法**: 
    1. 在側邊欄增加「+」按鈕。
    2. 實作新增時的對話框輸入檔名。
    3. 串接後端 API 並在完成後重整 `FileTree`。
- **驗證方式**: 使用者能在介面上完整執行：新增檔案 -> 編輯 -> 儲存 -> 刪除。
- **風險 / 備註**: 刪除操作應有二次確認畫面。
